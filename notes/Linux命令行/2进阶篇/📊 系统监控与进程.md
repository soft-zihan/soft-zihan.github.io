### 1. ps / top / htop / fastfetch - 进程与系统监控
- `ps -ef`：标准格式查看所有进程 (Process Status)
- `ps aux`：BSD格式查看进程 (含CPU/内存占用)
- `top`：实时显示系统资源使用情况 (P:按CPU，M:按内存，q:退出)
> 💡 **Pro Tip**: 尝试使用 **`btop`** 或 **`glances`** 替代 `top`，界面更酷炫，支持鼠标操作，显示更多硬件信息。
- `htop`：更直观的交互式进程监控，支持鼠标操作和更丰富的列展示
- `fastfetch`：现代化的系统信息展示工具 (类似 screenfetch/neofetch)，快速查看发行版、内核、硬件等信息

### 2. kill / pkill / killall - 终止进程
- `kill PID`：发送终止信号给指定PID
- `kill -9 PID`：强制杀死进程 (SIGKILL)
- `pkill 进程名`：按名称批量杀死进程
- `killall 进程名`：同 `pkill`，按名称杀死所有匹配进程

### 3. df / du / ncdu / lsblk - 磁盘与文件系统
- `df -h`：查看磁盘分区空间占用 (Disk Free)
- `du -sh`：查看当前目录总大小 (Disk Usage)
> 💡 **Pro Tip**: 尝试使用 **`dust`** 替代 `du`，提供更直观的树状磁盘占用图。
- `du -sh *`：查看当前目录下每个文件/文件夹的大小
- `ncdu`：交互式磁盘占用分析工具，比 `du` 更直观方便
- `lsblk`：列出所有块设备 (硬盘、分区) 的树状结构及挂载点

### 4. free / uptime - 系统资源
- `free -h`：查看内存使用情况
- `uptime`：查看系统运行时间及负载

### 5. iostat / vmstat - 性能分析
- `iostat`：查看磁盘I/O (Input/Output Statistics)
- `vmstat`：查看虚拟内存、CPU活动 (Virtual Memory Statistics)

### 6. dmesg / journalctl - 系统日志
- `dmesg`：显示内核环形缓冲区的信息，常用于诊断硬件或驱动问题。
- `journalctl -u nginx.service`：查看特定服务 (如 nginx) 的日志 (适用于 systemd 系统)。
- `journalctl -f`：实时滚动显示所有日志。

### 7. 任务管理与后台运行
- `&`：在命令末尾添加，使命令在后台运行 (如 `python server.py &`)
- `Ctrl + Z`：挂起当前前台任务到后台 (暂停状态)
- `jobs`：查看当前终端的所有后台任务
- `fg %n`：将编号为 n 的任务调回前台 (foreground)
- `bg %n`：让后台挂起的任务继续在后台运行 (background)
- `nohup 命令 &`：让命令在退出终端后继续运行 (No Hang Up)
- `disown %n`：将后台任务与终端解绑，效果类似 `nohup`

### 8. tmux - 终端复用器 (神器)
- `tmux new -s 名字`：启动一个新的 session
- `tmux attach`：重新连接到上一个 session
- `tmux ls`：列出所有 session
- `tmux kill-session -t 名字`：杀死 session
- **常用快捷键** (默认前缀为 `Ctrl + B`):
  - `d`：分离 (Detach) 当前 session (后台运行)
  - `c`：创建新窗口 (window)
  - `n/p`：切换到下/上一个窗口
  - `"` / `%`：水平 / 垂直分屏 (pane)
  - `方向键`：在分屏的窗格间导航

### 9. watch - 周期性执行
- `watch -n 1 'kubectl get pods'`：每秒刷新一次命令结果，实时监控变化。
- `watch -d 'ls -l'`：高亮显示变化的部分。
